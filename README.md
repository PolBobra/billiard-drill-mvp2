# Бильярд Тренер — инструкция по запуску

## Что уже готово
- Регистрация / вход (Supabase Auth)
- Дашборд игрока со статистикой навыков
- Страница "Зафиксировать удар" — выбираешь тип ошибки, угол, дистанцию → система находит подходящее упражнение из базы
- Страница со всей базой упражнений и фильтрами
- Готовая схема базы данных с 4 примерами упражнений

## Шаг 1. Supabase (база данных)
1. Зайди на supabase.com → New Project (через GitHub)
2. Дай проекту имя, придумай пароль для базы (сохрани его куда-нибудь)
3. Подожди 1-2 минуты, пока проект создастся
4. Слева в меню открой **SQL Editor** → New query
5. Открой файл `supabase/schema.sql` из этого проекта, скопируй всё содержимое, вставь в редактор → нажми **Run**
   - Это создаст все таблицы, права доступа и добавит 4 тестовых упражнения
6. Слева открой **Project Settings → API**
   - Скопируй **Project URL** и **anon public key** — они понадобятся на шаге 3

## Шаг 2. Залить код на GitHub
1. Создай новый пустой репозиторий на github.com (например, `billiard-training`)
2. В терминале, в папке проекта:
```
git init
git add .
git commit -m "первая версия"
git branch -M main
git remote add origin https://github.com/ТВОЙ_НИК/billiard-training.git
git push -u origin main
```

## Шаг 3. Vercel (хостинг)
1. Зайди на vercel.com → Sign in with GitHub
2. Add New → Project → выбери репозиторий `billiard-training`
3. В разделе **Environment Variables** добавь два значения из Шага 1:
   - `NEXT_PUBLIC_SUPABASE_URL` = твой Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = твой anon public key
4. Нажми **Deploy**
5. Через 1-2 минуты сайт будет доступен по ссылке вида `billiard-training-xxxx.vercel.app`

## Как добавлять новые упражнения
Пока — через Supabase: **Table Editor → exercises → Insert row**.
Позже я сделаю отдельную страницу-админку внутри сайта, чтобы делать это без захода в Supabase.

## Структура полей упражнения
- `name` — название
- `error_type` — тип ошибки (промах / недокрут / перекрут / слабый_удар / сильный_удар / плохой_выход / ошибка_позиции)
- `angle_min`, `angle_max` — диапазон угла удара в градусах
- `distance` — close / medium / far
- `situation_type` — тип ситуации (свободное поле, например angled_shot)
- `description` — описание задания
- `reps` — количество повторений
- `success_criteria` — критерий успеха (текст, например "10/15 попаданий")
- `difficulty` — сложность от 0 до 1
